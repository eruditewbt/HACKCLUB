Module 12: Project Showcase and Portfolio Packaging

Goal: convert completed course work into a portfolio-ready package with clear impact, reproducibility, and engineering quality evidence.

What To Showcase

  • One strong capstone with architecture diagram and execution flow.
  • One classical AI artifact (search/planning/value iteration) with correctness notes.
  • One ML artifact with evaluation table and error analysis.
  • One LLM artifact showing prompt design, schema constraints, and safety checks.

Portfolio Packaging Template

project/
  README.md
  docs/
    architecture.md
    evaluation.md
    risks_and_safeguards.md
  src/
  tests/
  run.ps1
  requirements.txt

Evidence Checklist

  • Problem statement and constraints are explicit.
  • Data assumptions and limitations are documented.
  • Model/algorithm choice is justified against alternatives.
  • Metrics and qualitative failure examples are included.
  • Reproduction steps run on a clean environment.

Run Guide

python AI/robot_planning_ai.py
python AI/course_snippets/model_eval_template.py
python AI/course_snippets/llm_prompt_with_schema.py

Completion Checklist

  • At least one end-to-end project is packaged with tests and docs.
  • A portfolio README includes architecture, metrics, and known risks.
  • All commands in the run guide execute successfully.
